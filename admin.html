Ersteinrichtung

GitHub Token wird AES-256-GCM verschlüsselt im Repo gespeichert

So funktioniert es:
Der GitHub-Token wird mit einem App-Passwort verschlüsselt (AES-256-GCM) und als config.enc im Repo gespeichert. Im Repo steht nur unlesbarer Chiffretext.

Jeder Benutzer mit Edit-Rechten (Editor, Admin, Owner) gibt beim Login einmalig das App-Passwort ein — danach läuft alles automatisch.

1. Gehe zu github.com → Settings → Tokens
2. Neuen Token erstellen: Scope repo aktivieren
3. Token + ein App-Passwort festlegen (z.B. shard2024)
4. App-Passwort an alle Admins/Editoren weitergeben
Wird sofort verschlüsselt — niemals im Klartext gespeichert
Dieses Passwort gilt für alle Benutzer mit Edit-Rechten — nicht dein persönliches Login-Passwort. Einmalig festlegen und sicher teilen.
